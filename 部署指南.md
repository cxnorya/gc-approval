# 公出审批系统部署指南（零基础版）

本指南按“GitHub Pages 前端 + Cloudflare 加速 + 阿里云轻量应用服务器后端 + MySQL”的方案编写。

下面示例域名请替换成你自己的：

```text
app.example.com   前端页面域名
api.example.com   后端接口域名
```

## 1. 准备账号和材料

需要准备：

- GitHub 账号：放前端代码，并启用 GitHub Pages
- Cloudflare 账号：管理域名 DNS 和加速
- 阿里云轻量应用服务器：运行后端和 MySQL
- 一个域名：例如 `example.com`
- 钉钉企业内部应用参数：可后面再填

## 2. 购买阿里云轻量应用服务器

建议配置：

```text
系统：Ubuntu 22.04
CPU/内存：2 核 2G 起步
带宽：3M 起步
磁盘：40G 起步
```

购买后，在阿里云控制台找到服务器公网 IP，例如：

```text
1.2.3.4
```

## 3. 放行服务器端口

在阿里云轻量应用服务器控制台，进入“防火墙”或“安全组”，放行：

```text
22    SSH 登录服务器
80    HTTP，用于申请证书和跳转 HTTPS
443   HTTPS，正式访问
```

注意：不建议对公网开放 `3000` 和 `3306`。

## 4. 登录服务器

在 Windows PowerShell 执行：

```bash
ssh root@你的服务器公网IP
```

示例：

```bash
ssh root@1.2.3.4
```

第一次登录会询问是否信任服务器，输入：

```bash
yes
```

## 5. 更新系统

```bash
apt update
```

说明：刷新 Ubuntu 软件列表。

```bash
apt upgrade -y
```

说明：升级已安装的软件包，`-y` 表示自动确认。

## 6. 安装 Node.js 18

```bash
curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
```

说明：添加 Node.js 18 的安装源。

```bash
apt install -y nodejs
```

说明：安装 Node.js 和 npm。

```bash
node -v
npm -v
```

说明：检查版本，能显示版本号就成功。

## 7. 安装 MySQL

```bash
apt install -y mysql-server
```

说明：安装 MySQL 数据库。

```bash
systemctl enable mysql
systemctl start mysql
```

说明：设置 MySQL 开机启动，并立即启动。

```bash
mysql -u root
```

说明：进入 MySQL 命令行。

进入 MySQL 后执行下面 SQL。请把 `你的强密码` 改成自己的数据库密码：

```sql
CREATE DATABASE IF NOT EXISTS gc_approval DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER IF NOT EXISTS 'gc_user'@'localhost' IDENTIFIED BY '你的强密码';
GRANT ALL PRIVILEGES ON gc_approval.* TO 'gc_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

说明：

- `gc_approval` 是系统数据库名
- `gc_user` 是后端连接数据库用的账号
- 不使用 root 账号跑业务更安全

## 8. 上传或克隆项目代码

推荐把代码上传到 GitHub，然后服务器用 git 拉取。

先安装 git：

```bash
apt install -y git
```

创建项目目录：

```bash
mkdir -p /opt/app
cd /opt/app
```

克隆你的仓库：

```bash
git clone https://github.com/你的用户名/你的仓库名.git gc-approval
```

进入项目：

```bash
cd /opt/app/gc-approval
```

## 9. 初始化数据库表

执行：

```bash
mysql -u gc_user -p gc_approval < database/init.sql
```

说明：

- `-u gc_user` 表示使用 `gc_user` 登录
- `-p` 表示接下来输入密码
- `< database/init.sql` 表示把初始化脚本导入 MySQL

如果没有报错，就说明建表成功。

## 10. 配置后端环境变量

进入后端目录：

```bash
cd /opt/app/gc-approval/backend
```

创建 `.env` 文件：

```bash
nano .env
```

粘贴下面内容，并按注释替换：

```env
NODE_ENV=production
PORT=3000

# 前端访问地址，用于钉钉消息跳转
FRONTEND_URL=https://app.example.com

# 允许访问后端的前端域名。多个域名用英文逗号分隔。
CORS_ORIGINS=https://app.example.com,https://你的用户名.github.io

# MySQL 数据库配置
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=gc_user
DB_PASSWORD=你的强密码
DB_NAME=gc_approval

# 钉钉配置。暂时不用钉钉时可以先留空。
DINGTALK_APP_KEY=
DINGTALK_APP_SECRET=
DINGTALK_AGENT_ID=
DINGTALK_CORP_ID=
DINGTALK_ROBOT_WEBHOOK=

# JWT 密钥。请改成一串很长的随机字符，不要用示例值。
JWT_SECRET=change_me_to_a_long_random_string
JWT_EXPIRES_IN=7d
```

保存 nano：

```text
按 Ctrl + O 保存
按 Enter 确认
按 Ctrl + X 退出
```

安装 PM2：




## 13. 安装 Nginx

```bash
apt install -y nginx
```

说明：Nginx 负责把公网 HTTPS 请求转发给本机 `3000` 端口的 Node.js 后端。

创建配置文件：

```bash
nano /etc/nginx/sites-available/gc-approval
```

先粘贴 HTTP 配置。把 `api.example.com` 改成你的后端域名：

```nginx
server {
    listen 80;
    server_name api.example.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用配置：

```bash
ln -s /etc/nginx/sites-available/gc-approval /etc/nginx/sites-enabled/gc-approval
```

检查配置：

```bash
nginx -t
```

重新加载 Nginx：

```bash
systemctl reload nginx
```

## 14. 配置 Cloudflare DNS

进入 Cloudflare 域名 DNS 页面，添加：

```text
类型：A
名称：api
内容：你的阿里云服务器公网 IP
代理状态：开启，也就是橙色云
```

如果你的前端也用自定义域名，再添加：

```text
类型：CNAME
名称：app
内容：你的用户名.github.io
代理状态：开启，也就是橙色云
```

Cloudflare SSL/TLS 模式建议选择：

```text
Full
```

## 15. 给后端申请 HTTPS 证书

安装 certbot：

```bash
apt install -y certbot python3-certbot-nginx
```

申请证书。把 `api.example.com` 改成你的后端域名：

```bash
certbot --nginx -d api.example.com
```

按提示输入邮箱，同意协议即可。

申请成功后测试：

```bash
curl https://api.example.com/api/health
```

正常返回：

```json
{"success":true,"message":"ok"}
```

## 18. 发布到 GitHub Pages

推荐做法：把整个项目推到 GitHub，然后在仓库设置里启用 Pages。

如果只想发布前端构建产物，可以用 `gh-pages` 分支：

```bash
cd E:\Codex\GC\frontend
git init
git add .
git commit -m "deploy frontend"
git branch -M main
git remote add origin https://github.com/你的用户名/你的仓库名.git
git push -u origin main
```

进入 GitHub 仓库：

```text
Settings -> Pages
```

选择：

```text
Source: Deploy from a branch
Branch: main
Folder: /dist
```

如果 GitHub 不允许直接选 `/dist`，可以改用 GitHub Actions 自动部署。

## 19. GitHub Pages 自定义域名

如果要使用 `app.example.com`：

1. 在 GitHub Pages 设置里填写 Custom domain：`app.example.com`
2. 在 Cloudflare DNS 添加 `app` 的 CNAME 到 `你的用户名.github.io`
3. 等待 DNS 生效
4. 勾选 GitHub Pages 的 Enforce HTTPS

## 22. 常见问题

### 前端打开后接口报错

检查 `frontend/.env.production`：

```env
VITE_API_BASE_URL=https://api.example.com/api
```

重新构建并发布前端：

```bash
cd E:\Codex\GC\frontend
npm run build
```

### 浏览器提示 CORS 跨域错误

检查服务器后端 `.env`：

```env
CORS_ORIGINS=https://app.example.com,https://你的用户名.github.io
```

修改后重启：

```bash
pm2 restart gc-approval
```

### 附件上传失败

检查后端是否正常：

```bash
curl https://api.example.com/api/health
```

检查 Nginx 是否允许上传较大文件。编辑：

```bash
nano /etc/nginx/nginx.conf
```

在 `http { ... }` 内添加：

```nginx
client_max_body_size 20m;
```

然后执行：

```bash
nginx -t
systemctl reload nginx
```

### 后端启动失败

查看日志：

```bash
pm2 logs gc-approval
```

常见原因：

- `.env` 数据库密码写错
- MySQL 没启动
- 端口 3000 被占用
- `JWT_SECRET` 没有填写

检查 MySQL：

```bash
systemctl status mysql
```

检查端口：

```bash
ss -lntp | grep 3000
```

### Cloudflare 访问不稳定

检查 Cloudflare：

- DNS 记录是否正确
- SSL/TLS 是否为 Full
- `api.example.com` 是否能访问 `/api/health`
- 阿里云安全组是否放行 80 和 443

## 23. 上线前检查清单

- `https://api.example.com/api/health` 返回 ok
- `frontend/.env.production` 已改成真实 API 域名
- `backend/.env` 里的 `FRONTEND_URL` 是真实前端域名
- `backend/.env` 里的 `CORS_ORIGINS` 包含真实前端域名
- Cloudflare SSL/TLS 模式是 Full
- 阿里云只开放 22、80、443
- 能用 `superadmin` 登录
- 能创建申请
- 能上传附件
- 能审批申请
