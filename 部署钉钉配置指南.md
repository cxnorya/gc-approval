# 公出审批系统 - 部署与钉钉配置指南

## 一、整体架构

```
用户手机/电脑钉钉
        ↓
   Cloudflare (加速)
        ↓
┌─────────────────┐
│  GitHub Pages   │ ← 前端静态页面
│  (前端托管)      │
└────────┬────────┘
         ↓ API请求
┌─────────────────┐
│  阿里云轻量服务器 │ ← 后端API + MySQL数据库
│  (后端托管)      │
└─────────────────┘
         ↓
   钉钉开放平台
```

## 二、第一步：部署后端到阿里云轻量服务器

### 2.1 准备工作

你需要准备：
- 阿里云轻量服务器（已安装 Linux 系统，推荐 Ubuntu 22.04）
- 服务器的公网 IP 地址
- 服务器的 root 密码或 SSH 密钥

### 2.2 连接服务器

1. 打开 Windows 的 PowerShell 或 终端
2. 输入以下命令连接（把 `你的服务器IP` 换成实际IP）：
```bash
ssh root@你的服务器IP
```
3. 输入密码登录

### 2.3 安装 Node.js 和 MySQL

在服务器上依次执行以下命令：

```bash
# 更新系统
apt update && apt upgrade -y

# 安装 Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# 验证安装
node -v
npm -v

# 安装 MySQL
apt install -y mysql-server

# 启动 MySQL
systemctl start mysql
systemctl enable mysql
```

### 2.4 配置 MySQL

```bash
# 进入 MySQL
mysql
```

在 MySQL 里执行：
```sql
-- 创建数据库
CREATE DATABASE gc_approval CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建用户（把 你的密码 换成你自己的密码）
CREATE USER 'gc_user'@'localhost' IDENTIFIED BY '你的密码';

-- 授权
GRANT ALL PRIVILEGES ON gc_approval.* TO 'gc_user'@'localhost';
FLUSH PRIVILEGES;

-- 退出
EXIT;
```

### 2.5 上传后端代码到服务器

**方法一：用 Git（推荐）**

在你自己电脑上，把代码推送到 GitHub，然后在服务器上拉取：

```bash
# 在服务器上
cd /opt
git clone 你的GitHub仓库地址.git
cd GC/backend
```

**方法二：用 scp 上传**

在你自己电脑的 PowerShell 里执行：
```bash
scp -r backend root@你的服务器IP:/opt/gc-backend
```

### 2.6 配置后端环境变量

在服务器上进入后端目录：
```bash
cd /opt/gc-backend  # 或者你的后端目录
cp .env.example .env  # 如果有示例文件的话
nano .env
```

把以下内容填进去（按你的实际情况修改）：
```env
NODE_ENV=production
PORT=3000

# 前端地址（GitHub Pages的地址）
CORS_ORIGIN=https://你的GitHub用户名.github.io/gc-approval

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=gc_user
DB_PASSWORD=你的数据库密码
DB_NAME=gc_approval

# 钉钉配置（从钉钉开放平台获取）
DINGTALK_APP_KEY=dingqqlv7iu5qprk866a
DINGTALK_APP_SECRET=BqrOfTeI9B-yctmjTlsmQtG1PhUUB1KTs-n9lVCuizj-mXvwfV0FK2H0YuwiVH1e

# JWT配置（可以不改，也可以改复杂点）
JWT_SECRET=gc_approval_jwt_secret_key_change_this_2024
JWT_EXPIRES_IN=7d
```

按 `Ctrl+O` 保存，`Ctrl+X` 退出。

### 2.7 安装依赖并启动

```bash
# 安装依赖
npm install

# 安装 PM2（进程管理，让后端一直在后台运行）
npm install -g pm2

# 启动后端
pm2 start src/app.js --name gc-backend

# 设置开机自启
pm2 startup
pm2 save
```

### 2.8 配置 Nginx（反向代理 + HTTPS）

```bash
# 安装 Nginx
apt install -y nginx
```

创建配置文件：
```bash
nano /etc/nginx/sites-available/gc-api
```

填入以下内容（把 `api.yourdomain.com` 换成你的域名）：
```nginx
server {
    listen 80;
    server_name api.yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

启用配置：
```bash
ln -s /etc/nginx/sites-available/gc-api /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```

### 2.9 配置 HTTPS（用 Cloudflare）

因为你用 Cloudflare 加速，HTTPS 可以直接在 Cloudflare 里配置：
1. 在 Cloudflare 把你的域名解析指向阿里云服务器IP
2. 在 Cloudflare 的 SSL/TLS 设置里选择 "Flexible" 或 "Full"
3. 这样就自动有 HTTPS 了

## 三、第二步：部署前端到 GitHub Pages

### 3.1 修改前端配置

在你自己电脑上，修改 `frontend/.env.production` 文件：

```env
# 后端API地址（你的阿里云服务器地址，要加 /api）
VITE_API_BASE_URL=https://api.yourdomain.com/api

# 部署目标是 GitHub
VITE_DEPLOY_TARGET=github
```

### 3.2 构建前端

```bash
cd frontend
npm install
npm run build
```

构建完成后，会生成一个 `dist` 文件夹，这就是要部署的静态文件。

### 3.3 部署到 GitHub Pages

**方法一：手动部署**

1. 在 GitHub 新建一个仓库，名字叫 `gc-approval`
2. 把 `dist` 文件夹里的所有文件上传到这个仓库
3. 进入仓库的 Settings → Pages
4. Source 选择 `main` 分支，根目录
5. 等几分钟，访问地址就是：`https://你的用户名.github.io/gc-approval/`

**方法二：用 gh-pages 工具自动部署**

在 `frontend/package.json` 里添加：
```json
{
  "scripts": {
    "deploy": "gh-pages -d dist"
  }
}
```

然后执行：
```bash
npm install --save-dev gh-pages
npm run build
npm run deploy
```

### 3.4 配置 Cloudflare 加速（可选）

如果你想用自己的域名而不是 github.io：

1. 在 Cloudflare 添加一个 CNAME 记录，比如 `app.yourdomain.com` 指向 `你的用户名.github.io`
2. 在 GitHub 仓库的 Pages 设置里，Custom domain 填上 `app.yourdomain.com`
3. 这样就可以通过 `https://app.yourdomain.com` 访问了

## 四、第三步：钉钉后台配置

### 4.1 配置首页地址

进入钉钉开发者后台 → 你的应用 → 网页应用：

| 配置项 | 填什么 | 说明 |
|--------|--------|------|
| **移动端首页地址** | `https://你的前端地址/` | 比如 `https://app.yourdomain.com/` |
| **PC端首页地址** | `https://你的前端地址/` | 和移动端一样 |

⚠️ **重要提醒**：地址必须是 HTTPS 的，钉钉只允许 HTTPS 地址。

### 4.2 配置权限

在钉钉开发者后台 → 你的应用 → 权限管理：

需要开通以下权限：
- ✅ 通讯录部门信息读取
- ✅ 通讯录用户信息读取
- ✅ 成员信息读取

### 4.3 配置服务器出口IP（可选）

如果钉钉要求配置服务器出口IP，就把你阿里云服务器的公网IP填进去。

## 五、第四步：验证钉钉免登流程

### 5.1 第一步：同步钉钉用户

1. 在钉钉里打开应用
2. 如果提示"用户不存在"，说明还没同步用户
3. 用管理员账号登录（在普通浏览器里登录）
4. 进入"管理" → "用户管理" → 点击"同步钉钉用户"
5. 等待同步完成

### 5.2 第二步：设置用户角色

1. 同步完成后，用户列表里会出现所有钉钉用户
2. 找到需要设置为"审批人"或"管理员"的用户
3. 调整他们的角色

### 5.3 第三步：测试免登

1. 打开钉钉手机端或PC端
2. 进入"工作台"，找到你的应用
3. 点击进入，应该能自动登录并跳转到申请页面

### 5.4 常见问题排查

**问题1：打开后显示"用户不存在"**
→ 先同步钉钉用户，确保用户在数据库里

**问题2：打开后一直转圈**
→ 检查前端控制台有没有报错
→ 检查后端日志有没有报错（`pm2 logs gc-backend`）

**问题3：提示跨域错误**
→ 检查后端 `.env` 里的 `CORS_ORIGIN` 是否正确
→ 注意地址最后不要有 `/`

**问题4：钉钉里打开空白**
→ 先用普通浏览器打开地址，确认页面正常
→ 检查是不是 HTTPS 证书问题

## 七、Cloudflare 配置建议

### 7.1 域名解析
- 前端域名（如 app.yourdomain.com）→ CNAME 指向 GitHub Pages
- 后端域名（如 api.yourdomain.com）→ A 记录指向阿里云服务器IP

### 7.2 SSL/TLS 设置
- 加密模式：Full（推荐）或 Flexible
- 始终使用 HTTPS：开启

### 7.3 速度优化
- Auto Minify：三个都开（JavaScript、CSS、HTML）
- Brotli：开启
- 火箭加载器：开启
