## 3. 数据库设计

### 3.1 用户表 (users)

```sql
CREATE TABLE users (
  id INT PRIMARY KEY AUTO_INCREMENT,
  dingtalk_userid VARCHAR(64) UNIQUE COMMENT '钉钉用户ID',
  name VARCHAR(100) NOT NULL COMMENT '姓名',
  department VARCHAR(100) COMMENT '部门',
  role ENUM('employee', 'approver', 'admin') DEFAULT 'employee' COMMENT '角色',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 3.2 公出地点表 (locations) - 可自定义管理

```sql
CREATE TABLE locations (
  id VARCHAR(10) PRIMARY KEY COMMENT '地点编号',
  name VARCHAR(100) NOT NULL COMMENT '地点名称',
  amount DECIMAL(10,2) NOT NULL COMMENT '标准金额(元/天)',
  status TINYINT DEFAULT 1 COMMENT '状态：0禁用 1启用',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 3.3 公出申请表 (applications)

```sql
CREATE TABLE applications (
  id INT PRIMARY KEY AUTO_INCREMENT,
  applicant_id INT NOT NULL COMMENT '申请人ID',
  approver_id INT NOT NULL COMMENT '审批人ID',
  apply_date DATE NOT NULL COMMENT '申请日期（可修改）',
  travel_date DATE NOT NULL COMMENT '出行日期',
  reason VARCHAR(500) NOT NULL COMMENT '公出事由',
  location_id VARCHAR(10) NOT NULL COMMENT '公出地点ID',
  person_count INT NOT NULL COMMENT '出行人数',
  amount DECIMAL(10,2) NOT NULL COMMENT '报销金额',
  status ENUM('draft', 'pending', 'approved', 'rejected', 'cancelled') DEFAULT 'draft',
  reject_reason VARCHAR(500) COMMENT '驳回原因',
  approver_comment VARCHAR(500) COMMENT '审批备注',
  approved_at TIMESTAMP NULL COMMENT '审批时间',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (applicant_id) REFERENCES users(id),
  FOREIGN KEY (approver_id) REFERENCES users(id),
  FOREIGN KEY (location_id) REFERENCES locations(id)
);
```

### 3.4 操作日志表 (operation_logs)

```sql
CREATE TABLE operation_logs (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,
  action VARCHAR(50) NOT NULL,
  target_type VARCHAR(50) NOT NULL,
  target_id INT NOT NULL,
  detail TEXT,
  ip_address VARCHAR(50),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 5. 报销金额计算逻辑

```javascript
// calculationService.js

/**
 * 计算报销金额
 * @param {number} personCount - 出行人数
 * @param {number} locationAmount - 公出地点标准金额
 * @returns {number} 报销金额
 */
function calculateAmount(personCount, locationAmount) {
  if (personCount <= 0) {
    throw new Error('出行人数必须大于0');
  }

  if (personCount === 1) {
    // 单人出行：金额 = 人数 × 地点金额
    return personCount * locationAmount;
  } else {
    // 多人出行：金额 = 人数 × 地点金额 × (1 - 人数 × 0.1)
    return personCount * locationAmount * (1 - personCount * 0.1);
  }
}

module.exports = { calculateAmount };
```

**公式说明**:
- 出行人数 = 1: 报销金额 = 1 × 地点金额
- 出行人数 ≥ 2: 报销金额 = 出行人数 × 地点金额 × (1 - 出行人数 × 0.1)

**示例计算**:
| 出行人数 | 地点金额 | 计算公式 | 报销金额 |
|---------|---------|---------|---------|
| 1 | 150 | 1 × 150 | 150元 |
| 2 | 150 | 2 × 150 × 0.9 | 270元 |
| 3 | 150 | 3 × 150 × 0.8 | 360元 |
| 5 | 150 | 5 × 150 × 0.6 | 450元 |
| 10 | 150 | 10 × 150 × 0.2 | 300元 |

## 6. 钉钉集成方案

### 6.1 钉钉应用配置

1. 创建企业内部应用
2. 配置应用权限（通讯录、个人信息）
3. 配置登录权限
4. 创建机器人用于消息通知

### 6.2 免登流程

```
1. 前端跳转到钉钉授权页
2. 用户授权
3. 钉钉返回code
4. 前端将code发送到后端
5. 后端使用code换取userid
6. 后端查询/创建用户信息
7. 返回JWT Token给前端
```

### 6.3 消息通知

使用钉钉机器人webhook推送审批通知：
- 新的待审批申请
- 审批结果通知
- 驳回原因通知

## 9. 部署方案

### 9.1 前端部署（阿里云OSS）

1. 打包前端项目: `npm run build`
2. 上传dist目录到OSS
3. 配置静态网站托管
4. 配置自定义域名（可选）
5. 开启HTTPS

### 9.2 后端部署（阿里云ECS）

1. 安装Node.js环境
2. 克隆/上传代码
3. 安装依赖: `npm install`
4. 配置环境变量
5. 启动服务: `pm2 start`
6. 配置Nginx反向代理
7. 配置SSL证书

### 9.3 数据库部署（阿里云RDS）

1. 创建MySQL实例
2. 创建数据库和用户
3. 执行初始化脚本
4. 配置白名单
