# 公出审批系统 PRD (产品需求文档)

## 1. 产品概述

- **产品名称**: 公出审批系统
- **产品类型**: 企业内部审批流程管理系统
- **核心功能**: 实现公出申请、审批、报销金额自动计算的数字化流程，对接钉钉平台
- **目标用户**: 企业员工、行政管理人员、审批负责人

## 2. 表单设计

### 2.1 公出申请表单

| 字段名称 | 字段类型 | 必填 | 说明 |
|---------|---------|------|------|
| 申请日期 | 日期选择器 | 是 | 默认当天，可修改 |
| 出行日期 | 日期选择器 | 是 | 计划公出日期 |
| 申请人 | 系统自动填充 | 是 | 当前登录用户 |
| 审批负责人 | 人员选择器 | 是 | 下拉选择 |
| 公出事由 | 文本输入 | 是 | 最少5个字符 |
| 公出地点 | 下拉选择 | 是 | 预设地点及对应金额（可自定义管理） |
| 出行人数 | 数字输入 | 是 | 最小1，最大50 |
| 报销金额 | 自动计算 | 是 | 根据公式自动计算 |

### 2.2 公出地点管理（管理员可自定义）

管理员可在系统设置中添加、编辑、删除公出地点及其对应金额。

**默认地点配置**:

| 地点编号 | 地点名称 | 标准金额(元/天) |
|---------|---------|----------------|
| A001 | 本市本区 | 100 |
| A002 | 本市外区 | 150 |
| A003 | 省内其他城市 | 250 |
| A004 | 省外一线城市 | 400 |
| A005 | 省外其他城市 | 300 |

### 2.3 报销金额计算公式

**规则1（单人出行）**: 出行人数 = 1时
```
报销金额 = 出行人数 × 公出地点对应金额
```

**规则2（多人出行）**: 出行人数 ≥ 2时
```
报销金额 = 出行人数 × 公出地点对应金额 × (1 - 出行人数 × 0.1)
```

**示例计算**:
- 1人前往A002地点: 1 × 150 = 150元
- 3人前往A002地点: 3 × 150 × (1 - 3 × 0.1) = 3 × 150 × 0.7 = 315元
- 5人前往A002地点: 5 × 150 × (1 - 5 × 0.1) = 5 × 150 × 0.5 = 375元

## 3. 用户角色

### 3.1 普通员工
- 提交公出申请
- 查看本人申请记录及状态
- 修改本人待审批的申请
- 取消本人申请

### 3.2 审批负责人
- 查看所有待审批申请
- 审批/驳回申请
- 查看已审批记录

### 3.3 系统管理员
- 所有审批负责人权限
- 公出地点及金额配置管理
- 用户管理
- 导出每月审批数据（Excel格式）
- 系统参数配置

## 4. 功能模块

### 4.1 用户认证模块
- 钉钉扫码登录
- 钉钉企业内部应用授权
- 用户信息同步

### 4.2 申请管理模块
- 新建公出申请
- 申请列表（支持按状态筛选）
- 申请详情查看
- 申请修改（仅待审批状态）
- 申请取消

### 4.3 审批流程模块
- 待审批列表
- 审批操作（通过/驳回）
- 审批历史记录

### 4.4 数据统计模块
- 月度审批数据统计
- 数据导出功能（Excel）
- 筛选条件：月份、审批状态

### 4.5 系统配置模块（管理员）
- 公出地点管理（增删改查）
- 金额标准配置
- 用户权限管理

## 5. 审批流程

```
[员工提交申请] → [系统自动校验] → [推送钉钉通知给审批人] → [审批人审批]
                                                                      ↓
                                                        [通过] ← → [驳回]
                                                          ↓         ↓
                                                    [更新状态]  [通知申请人]
```

## 6. 状态流转

| 状态码 | 状态名称 | 可执行操作 |
|-------|---------|----------|
| DRAFT | 草稿 | 提交 |
| PENDING | 待审批 | 审批/驳回/取消 |
| APPROVED | 已通过 | - |
| REJECTED | 已驳回 | 修改后重新提交 |
| CANCELLED | 已取消 | - |

## 7. 通知机制

- 提交申请后：通过钉钉机器人通知审批负责人
- 审批完成后：通过钉钉机器人通知申请人
- 驳回时：同时通知驳回原因

## 10. 技术对接

### 10.1 钉钉对接
- 钉钉企业内部应用
- 钉钉免登授权
- 钉钉机器人通知

### 10.2 部署架构
- 前端：静态页面托管（阿里云OSS或其他CDN）- 响应式设计，适配PC和移动端
- 后端：阿里云服务器（Node.js）
- 数据库：阿里云RDS MySQL
- 文件存储：阿里云OSS

## 11. 非功能性需求

- 响应时间：页面加载 < 2秒
- 数据安全：HTTPS加密传输
- 兼容性：Chrome、Firefox、Safari、Edge最新两个版本
- **移动端适配**：底部导航栏，支持手机端查看和审批
- **PC端适配**：顶部导航，侧边栏布局
